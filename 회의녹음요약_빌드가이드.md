# 회의녹음요약 앱 빌드 가이드

> 작성 기준: v3.0.3 / 2026-04-09

---

## 사전 준비 (최초 1회)

### 1. Python 설치 확인

Windows 검색창에서 `cmd` 실행 후 아래 명령어 입력:

```
python --version
```

Python 3.10 이상이 출력되면 정상. 미설치 시 https://python.org 에서 다운로드.

> ⚠️ 설치 시 **"Add Python to PATH"** 체크박스 반드시 체크

---

### 2. 필수 패키지 사전 설치 (선택사항)

빌드 전에 아래 명령어를 실행하면 빌드 속도가 빨라짐:

```
pip install pyinstaller google-genai sounddevice soundfile requests anthropic
```

---

## 빌드 절차 (본 작업)

### Step 1. 폴더 열기

탐색기에서 아래 경로로 이동:

```
회의녹음요약 폴더 (예: C:\Users\사용자명\회의녹음요약)
```

폴더 안에 아래 파일들이 있어야 함:
- `build.bat` ← 이 파일을 실행할 것
- `app_dist\gemini_service.py`
- `app_dist\main.py`
- `requirements.txt`

---

### Step 2. build.bat 실행

`build.bat` 파일을 **더블클릭** 또는 우클릭 → **관리자 권한으로 실행**

> 관리자 권한 실행을 권장. 일반 실행 시 패키지 설치 오류 발생할 수 있음.

콘솔 창이 열리면서 아래 단계가 자동 진행됨:

```
[1/4] 패키지 설치 중...     ← pip install 자동 실행 (30초~1분)
[2/4] 앱 아이콘 생성 중...  ← 아이콘 PNG → ICO 변환
[3/4] exe 빌드 중...         ← PyInstaller 빌드 (1~3분)
[4/4] 빌드 완료!
```

---

### Step 3. 빌드 완료 확인

빌드 완료 후 아래 경로에 exe 파일 생성됨:

```
회의녹음요약\dist\회의녹음요약.exe
```

콘솔 창에 아래 메시지가 나오면 성공:

```
================================================
 실행 파일 위치:
 C:\...\dist\회의녹음요약.exe
================================================
```

> Enter 키 또는 창 닫기 후 종료

---

### Step 4. 기존 앱 교체

| 단계 | 작업 |
|------|------|
| ① | 기존에 사용 중인 `회의녹음요약.exe` 가 있는 폴더로 이동 |
| ② | 기존 exe 파일 삭제 (또는 백업용으로 다른 이름으로 변경) |
| ③ | `dist\회의녹음요약.exe` 를 해당 폴더에 복사 |
| ④ | 새 exe 파일 실행하여 정상 동작 확인 |

## 오류 발생 시 조치

### 오류 1: "Python이 설치되어 있지 않습니다"

→ https://python.org 에서 Python 3.11 설치. 설치 시 **"Add Python to PATH"** 체크 필수.

---

### 오류 2: "패키지 설치 실패" / pip 오류

cmd 창을 관리자 권한으로 열고 아래 명령어 실행:

```
pip install --upgrade pip
pip install -r requirements.txt
pip install pyinstaller
```

---

### 오류 3: 빌드 중 "ModuleNotFoundError"

누락 모듈명을 확인 후 아래 명령어 실행 (예: `requests` 누락 시):

```
pip install requests
```

이후 `build.bat` 재실행.

---

### 오류 4: Windows SmartScreen 차단

빌드된 exe 실행 시 "Windows의 PC 보호" 메시지가 나타날 경우:
1. **"추가 정보"** 클릭
2. **"실행"** 클릭

> 코드 서명 인증서 미구매 시 항상 발생. 정상 동작에 문제 없음.

---

## 빌드 불필요한 경우

`app\gemini_service.py` 파일만 수정한 경우(프롬프트 텍스트 변경 등):

→ **Python으로 직접 실행** 시 재빌드 없이 바로 적용:

```
python app\main.py
```

exe 배포가 필요할 때만 `build.bat` 실행.

## 파일 구조 요약

```
회의녹음요약\
├── app_dist\           ← 빌드 소스 (수정 시 이 폴더 편집)
│   ├── main.py
│   ├── gemini_service.py   ← 회의록 프롬프트 관리
│   └── ...
├── app\                ← 직접 실행용 (gitignore 제외)
│   └── gemini_service.py   ← app_dist와 동기화 유지
├── dist\               ← 빌드 결과물 (배포용 exe)
│   └── 회의녹음요약.exe
├── build.bat           ← 빌드 실행 스크립트
└── requirements.txt
```

> `app_dist\gemini_service.py` 수정 → `build.bat` 실행 → `dist\회의녹음요약.exe` 교체
