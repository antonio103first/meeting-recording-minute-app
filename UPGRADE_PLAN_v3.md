## 1. 업그레이드 개요

현행 v2.0은 회의 녹음·STT 변환·AI 요약의 핵심 파이프라인을 구축하였다. v3.0은 **결과물 가독성**, **출력 다양화**, **UX 개선** 세 축을 중심으로 아래 6개 기능을 추가/개선한다.

| 항목 | 내용 | 유형 |
|------|------|------|
| F-01 | 회의목록 탭 — 요약/STT 분리 뷰 + 액션 버튼 추가 | 신규/개선 |
| F-02 | 출력 기능 — PDF 변환 / 네트워크 프린트 / 로컬 프린트 | 신규 |
| F-03 | 요약 방식 구조 개편 (전 방식 MD 출력 통일) | 개선 |
| F-04 | 흐름 중심 요약 방식 신규 추가 | 신규 |
| F-05 | 설정 탭 전면 개편 (요약방식·API설정·명칭 변경) | 개선 |
| F-06 | 녹음/변환 탭 — MP3 파일선택 + 2-트랙 회의록 변환 | 신규/개선 |

## 2. 기능 상세 명세

### F-01. 회의목록 탭 — 요약 / STT 분리 뷰

#### 현행
회의목록 탭에서 선택한 회의의 요약·STT 원문을 단일 뷰로 표시.

#### 변경

| 항목 | 세부 내용 |
|------|---------|
| 탭 구조 | 회의 상세 진입 시 **"회의록 요약"** / **"STT 변환 원문"** 2개 탭으로 분리 표시 |
| MD 표시 | 두 탭 모두 **Markdown 원문 그대로** 스크롤 표시 (렌더링 없이 MD 양식 유지) |
| 액션 버튼 | 각 탭 하단에 **전체보기 / 출력·인쇄 / 공유 / 삭제** 4개 버튼 제공 |
| 레이아웃 | 상단 탭바 + 중앙 스크롤 영역 + 하단 버튼바 구조 |

#### 화면 구조

```
┌──────────────────────────────────────────────────┐
│ [회의목록] > 2026-03-22_투자심사회의               │
├─────────────────┬────────────────────────────────┤
│   회의록 요약   │       STT 변환 원문             │  ← 탭 전환
├─────────────────┴────────────────────────────────┤
│                                                   │
│   (선택된 탭의 MD 원문 스크롤 표시)                │
│                                                   │
├──────────────────────────────────────────────────┤
│  [전체보기]  [출력/인쇄]  [공유 ▼]  [삭제]        │  ← 액션 버튼바
└──────────────────────────────────────────────────┘
```

#### 액션 버튼 상세

| 버튼 | 동작 |
|------|------|
| **전체보기** | 별도 최대화 창에서 전체 내용 표시 (복사 가능) |
| **출력/인쇄** | F-02 출력 기능 연계 — PDF 저장 / 네트워크 프린트 / 로컬 프린트 선택 |
| **공유 ▼** | 드롭다운: 카카오톡 공유 / 이메일 전송 / 클립보드 복사 |
| **삭제** | 확인 다이얼로그 후 해당 탭(요약 or STT) 파일 삭제 + DB 업데이트 |

#### 구현 범위
- `database.py` : `summaryText` / `sttText` 필드 분리 조회, 개별 삭제 처리
- `main.py` : 탭위젯, 전체보기 다이얼로그, 공유 드롭다운, 삭제 확인 다이얼로그
- 카카오톡 공유: KakaoTalk PC 앱 연동 또는 URL 스킴 호출
- 이메일: `smtplib` 또는 기본 메일 클라이언트 호출 (`mailto:` URI)

### F-02. 출력 기능 — PDF 변환 / 네트워크 프린트 / 로컬 프린트

#### 기능 설명
회의록 요약 탭에서 **"출력/인쇄"** 버튼 클릭 시 출력 방식을 선택하여 실행한다.

| 출력 방식 | 설명 | 구현 방법 |
|---------|------|---------|
| **PDF 저장** | MD → PDF 파일로 로컬 저장 | `reportlab` |
| **네트워크 프린트** | 네트워크(IP) 프린터로 직접 출력 | `win32print` + IP 프린터 설정 |
| **로컬 프린트** | PC에 연결된 로컬 프린터로 직접 출력 | `win32print` 프린터 목록 조회 |

#### 출력 레이아웃 — ⚠ Confirm 필요

### F-04. 흐름 중심 요약 방식 (신규)

#### 개념
회의가 **어떻게 전개되었는지**를 중심으로 요약한다. 발언자나 주제가 아닌, 논의의 **흐름과 맥락**을 따라가며 회의의 전체 서사를 재구성한다.

#### 목적
- 회의 참석자가 흐름을 빠르게 파악
- 비참석자가 회의 맥락·분위기·결정 배경을 이해
- 후속 회의에서 이전 회의 흐름 복기

---

#### 흐름 중심 요약 프롬프트

```
다음 회의 STT 텍스트를 "흐름 중심" 방식으로 요약해 주세요.

[흐름 중심 요약 규칙]
1. 회의를 흐름에 따라 3~6개의 단계로 구분하세요.
2. 각 단계는 다음 항목을 포함합니다:
   - 단계명 (예: "도입 및 안건 공유", "핵심 쟁점 논의", "합의 및 결정")
   - 해당 단계의 주요 내용 (2~5문장)
3. 각 단계 번호는 "1. / 2. / 3." 형식으로 표기하세요.
4. 시간(타임스탬프)은 표시하지 않습니다.
5. 마지막에 "최종 결론 및 후속 조치" 섹션을 표로 추가하세요.
6. 전체 분량은 A4 1~2페이지 수준으로 작성하세요.
7. 반드시 Markdown 형식으로 작성하세요.

[STT 원문]
{stt_text}
```

### F-06. 녹음/변환 탭 — 2-트랙 회의록 변환 구조

#### 개요
녹음/변환 탭을 **두 개의 독립 영역**으로 구성한다.

- **영역 A (STT 변환 결과)** : 기존 흐름 유지 — 녹음 후 변환 시작/중단, 설정의 요약 방식으로 자동 요약, 저장명 Confirm
- **영역 B (회의록 요약 결과)** : 신규 — 저장된 STT 파일 불러오기 후 별도 요약 방식 선택, 추가 변환·저장

---

#### 영역 A — STT 변환 결과 (기존 유지 + MP3 파일선택 추가)

| 버튼 | 동작 |
|------|------|
| **MP3 파일선택** | 로컬 저장된 MP3 파일을 파일 탐색기로 직접 선택 → 변환 대상으로 로드 (녹음 없이 파일 직접 변환 가능) |
| **변환 시작** | 녹음 파일 또는 선택된 MP3 파일로 STT 변환 실행 → 완료 시 설정에 지정된 요약 방식으로 자동 요약 |
| **중단** | STT 변환 또는 요약 진행 중 강제 중단 |

**저장 흐름 (기존 동일):**
```
[MP3 파일선택] (선택 시) → MP3 파일 로드
       ↓
변환 시작 → STT 완료 → 요약 API 요약 (설정 방식 자동 적용)
       ↓
저장 파일명 입력 다이얼로그
  파일명: [2026-03-22_투자심사회의          ]
  [저장]  [취소]
       ↓
로컬 저장 + DB 저장 + Google Drive 업로드 (설정 ON 시)
```

---

#### 영역 B — 회의록 요약 결과 (신규 추가)

| 버튼 | 동작 |
|------|------|
| **STT 파일 선택** | 로컬 저장된 `.md` STT 파일 불러오기 (파일 탐색기) |
| **회의록 변환** | 요약 방식 선택 팝업 → Gemini 요약 → 저장명 입력 → 추가 저장 |
| **중지** | 변환 진행 중 강제 중단 |

**추가 변환 흐름:**
```
[STT 파일 선택] → STT 파일 로드
       ↓
[회의록 변환] 버튼 클릭
       ↓
요약 방식 선택 팝업  ← 설정과 독립 선택
  ○ 화자 중심
  ○ 주제 중심
  ○ 회의양식
  ○ 강의 요약
  ● 흐름 중심  (초기값: 설정의 기본 요약 방식)
  [확인]  [취소]
       ↓
Gemini API 요약 실행 (중지 버튼으로 취소 가능)
       ↓
저장 파일명 입력 다이얼로그
  파일명: [2026-03-22_투자심사회의_흐름중심    ]
  [저장]  [취소]
       ↓
로컬 저장 (.md) + DB 추가 저장 + Google Drive 업로드 (설정 ON 시)
       ↓
회의목록 탭에 신규 항목으로 추가
```

#### F-06-S. 파일 저장 구조 검토

현행 v2.0은 회의 1건당 `MP3 + 통합 텍스트(STT+요약 혼합)` 2개 파일을 저장한다.
v3.0에서 STT와 요약을 분리 저장·추가 저장 기능이 추가됨에 따라 **파일 분리 구조로 전환**을 검토한다.

**현행 저장 구조:**
```
~/Documents/Meeting recording/
├── audio/        2026-03-22_투자심사.mp3
└── summary/      2026-03-22_투자심사.txt   ← STT + 요약 혼합 저장
```

**v3.0 권장 저장 구조:**
```
~/Documents/Meeting recording/
├── audio/        2026-03-22_투자심사.mp3         ← 녹음 원본
├── stt/          2026-03-22_투자심사_stt.md       ← STT 원문 단독 저장
└── summary/      2026-03-22_투자심사_화자중심.md   ← 요약 (방식명 suffix)
                  2026-03-22_투자심사_흐름중심.md   ← 동일 STT 다른 방식 추가 저장 가능
```

**DB 스키마 변경안:**

| 필드 | 현행 | v3.0 변경 |
|------|------|---------|
| `sttLocalPath` | 통합 파일 경로 | STT 단독 파일 경로 (`stt/` 폴더) |
| `summaryLocalPath` | 통합 파일 경로 | 요약 파일 경로 (`summary/` 폴더) |
| `summaryMode` | 없음 | 요약 방식명 저장 (신규 추가) |

**파일 분리의 이점:**

| 항목 | 설명 |
|------|------|
| 재활용성 | STT 파일 1개로 여러 방식 요약 생성 가능 (영역 B 지원) |
| 가독성 | 회의목록 탭에서 STT / 요약 탭 분리 표시 명확화 (F-01 연계) |
| 출력 | 요약 파일만 선택하여 인쇄 가능 (F-02 연계) |
| 관리 | 파일 탐색기에서 STT·요약 구분 명확 |

> **⚠ 파일 구조 변경 시 기존 저장 데이터 마이그레이션 로직 필요** — 기존 통합 파일에서 STT/요약 파싱 후 분리 저장 또는 신규 저장분부터 적용하는 방식 선택 필요.

#### 구현 범위
- `main.py` : 영역 A/B 분리 UI, STT 파일 선택 버튼, 회의록 변환 버튼, 중지 버튼, 요약 방식 팝업, 저장명 다이얼로그
- `gemini_service.py` : 취소 플래그(`cancel_flag`) 처리
- `database.py` : `summaryMode` 필드 추가, 파일명 중복 방지 suffix 처리
- `file_manager.py` : `stt/`, `summary/` 폴더 분리 저장 로직
