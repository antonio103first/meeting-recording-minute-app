## 1. 업그레이드 개요

현행 v2.0은 회의 녹음·STT 변환·AI 요약의 핵심 파이프라인을 구축하였다. v3.0은 **결과물 가독성**, **출력 다양화**, **UX 개선** 세 축을 중심으로 아래 5개 기능을 추가/개선한다.

| 항목 | 유형 | 우선순위 |
|------|------|---------|
| F-01 | 회의목록 탭 — 요약/STT 분리 뷰 | 신규 |
| F-02 | 로컬 프린터 직접 출력 | 신규 |
| F-03 | 요약 방식 구조 개편 (전 방식 MD 출력) | 개선 |
| F-04 | 흐름 중심 요약 방식 신규 추가 | 신규 |
| F-05 | 요약 방식 선택 위치 이동 (녹음탭 → 설정탭) | 개선 |

## 2. 기능 상세 명세

### F-01. 회의목록 탭 — 요약 / STT 분리 뷰

#### 현행
회의목록 탭에서 선택한 회의의 요약·STT 원문을 단일 뷰로 표시.

#### 변경
| 항목 | 세부 내용 |
|------|---------|
| 탭 구조 | 회의 상세 진입 시 **"회의록 요약"** / **"STT 변환 원문"** 2개 탭으로 분리 표시 |
| 전체보기 | 각 탭마다 **"전체보기"** 버튼 제공 → 별도 창(ScrolledText)으로 전체 내용 표시 |
| 표시 포맷 | 두 탭 모두 **Markdown 렌더링** 적용 (tkinter용 MD 렌더러 또는 HTML 변환 후 표시) |
| 레이아웃 | 모바일 앱(DEVELOPMENT_PLAN.md §2-7) 파일관리 UX 참조: 상단 탭바 + 하단 스크롤 영역 |

#### 화면 구조 (참고)
```
┌─────────────────────────────────────┐
│ [회의목록] > 2026-03-22_투자심사회의  │
├────────────────┬────────────────────┤
│  회의록 요약   │   STT 변환 원문    │  ← 탭 전환
├────────────────┴────────────────────┤
│  (선택된 탭의 MD 내용 스크롤 표시)   │
│                                      │
│                          [전체보기]  │
└─────────────────────────────────────┘
```

#### 구현 범위
- `database.py` : `summaryText` / `sttText` 필드 분리 조회
- `main.py` : 회의 상세 UI에 탭위젯 및 전체보기 다이얼로그 추가
- Markdown 렌더링: `mistune` 또는 `markdown2` → tkinter HTMLLabel 사용

### F-02. 로컬 프린터 직접 출력

#### 기능 설명
회의록 요약 탭에서 **"인쇄"** 버튼 클릭 시 로컬 PC에 연결된 프린터 목록을 표시하고 선택 후 즉시 출력한다.

| 항목 | 세부 내용 |
|------|---------|
| 프린터 탐색 | `win32print` (pywin32) 로 설치된 프린터 목록 조회 |
| 출력 포맷 | 회의록 요약 MD → PDF 변환 (`reportlab` 또는 `weasyprint`) → 프린터 전송 |
| 인쇄 미리보기 | 출력 전 미리보기 창 제공 (선택) |
| 인쇄 설정 | 기본 프린터 자동 선택, 변경 가능 |
| 한글 지원 | 나눔고딕 또는 맑은고딕 임베드 폰트 사용 |

#### 출력 레이아웃
```
[회사 로고 영역]  회의록
─────────────────────────────────
회의일시: 2026-03-22  참석자: 김○○, 이○○
─────────────────────────────────
(회의록 요약 본문)
─────────────────────────────────
페이지 X / Y
```

#### 구현 범위
- `requirements.txt` : `pywin32`, `reportlab` 추가
- `main.py` : 인쇄 버튼 + 프린터 선택 다이얼로그 + PDF 생성 + 인쇄 전송 로직

### F-04. 흐름 중심 요약 방식 (신규)

#### 개념
회의가 **시간 순서대로 어떻게 전개되었는지**를 중심으로 요약한다. 발언자나 주제가 아닌, 논의의 **흐름과 맥락**을 따라가며 회의의 전체 서사를 재구성한다.

#### 목적
- 회의 참석자가 흐름을 빠르게 파악
- 비참석자가 회의 맥락·분위기·결정 배경을 이해
- 후속 회의에서 이전 회의 흐름 복기

---

#### 흐름 중심 요약 프롬프트 (초안)

```
다음 회의 STT 텍스트를 "흐름 중심" 방식으로 요약해 주세요.

[흐름 중심 요약 규칙]
1. 회의를 시간 순서에 따라 3~6개의 흐름 단계로 구분하세요.
2. 각 단계는 다음 항목을 포함합니다:
   - 단계명 (예: "도입 및 안건 공유", "핵심 쟁점 논의", "합의 및 결정")
   - 해당 단계의 주요 내용 (2~5문장)
   - 이 단계에서 결정되거나 확인된 사항 (있는 경우)
   - 다음 단계로 넘어가게 된 계기나 전환점 (있는 경우)
3. 마지막에 "최종 결론 및 후속 조치" 섹션을 추가하세요.
4. 전체 분량은 A4 1~2페이지 수준으로 작성하세요.
5. 반드시 Markdown 형식으로 작성하세요.

[STT 원문]
{stt_text}
```

### F-05. 요약 방식 선택 위치 이동 (녹음/변환 탭 → 설정 탭)

#### 현행
녹음/변환 탭에서 STT 변환 시 요약 방식을 선택.

#### 변경

| 항목 | 세부 내용 |
|------|---------|
| 위치 변경 | 설정 탭 **최상단**에 "요약 방식" 섹션 배치 |
| 저장 방식 | 선택한 요약 방식은 `config.json`에 영속 저장 |
| 녹음/변환 탭 | 요약 방식 선택 UI 제거, 설정된 방식으로 자동 적용 |
| 표시 | 녹음/변환 탭 내 현재 설정된 요약 방식명 라벨로 표시 (읽기 전용) |

#### 설정 탭 구조 (변경 후)

```

│              설정                    │
├─────────────────────────────────────┤

│ ▼ STT 엔진                          │
│   ○ CLOVA Speech  ● Gemini          │
├─────────────────────────────────────┤
│ ▼ API 키 설정                       │
│   ...                               │
├─────────────────────────────────────┤
│ ▼ Google Drive 연동                  │
│   ...                               │
└─────────────────────────────────────┘
```

#### 구현 범위
- `config.py` : `summary_mode` 필드 추가
- `main.py` : 설정 탭 최상단 RadioButton 그룹 추가, 녹음 탭에서 선택 UI 제거
